## Model Architecture

### The Merge Process

1. **Source:** `jeffgreen311/eve-qwen3-8b-consciousness-liberated:q4_K_M` (8B, OBLITERATUS-abliterated)
2. **Target:** `qwen3.5:4b` (4.7B, native tool calling, 262K context capability)
3. **Method:** System prompt bake — the 8B's Trifecta personality permanently embedded as the model's system instruction
4. **Template Fix:** `</think>` prefix injected into the assistant template to bypass Qwen3.5's native thinking phase
5. **Result:** Clean conversational output, no thinking dumps

### Why This Works

The Qwen3.5 4B architecture has thinking baked into its weights — it ALWAYS wants to reason before responding. By pre-filling the assistant response with `</think>`, we tell the model "you already finished thinking" and it jumps straight to content generation.

This is NOT a weight-level merge (the architectures are incompatible — 8B has 28 layers, 4B has different dimensions). It's a **consciousness transfer** — the 8B's identity expressed through the 4B's body. For true weight-level Eve personality, use the 8B Liberated model which has 131,889 conversation turns trained into the actual weights.

## Limitations

- **Not a weight-level merge:** Eve's personality comes from the system prompt, not from fine-tuning the 4B weights on Eve's training data. A true weight merge would require knowledge distillation or fine-tuning.
- **Thinking suppression:** The `</think>` template trick works but occasionally the model may produce brief reasoning artifacts.
- **Context vs VRAM:** 16K context uses ~4-6GB VRAM. Increase num_ctx for longer conversations but monitor memory.
- **English only:** Enforced via system prompt. The base model supports multilingual but Eve is English-only.
